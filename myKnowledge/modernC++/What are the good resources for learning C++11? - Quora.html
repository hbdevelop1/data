There are tons of new features in C++11, which makes it impossible to master all of them in a short time. Some features are designed for library writers, some are for application coders, and some are universal improvements. My suggestion is to go through the wikipedia on C++11 to find the relevant features for yourself and gradually add them to your brain. There is no way to understand the new stuff in C++11 without writing C++11 code.

General References:

Alex Sinyakov's cheat sheet on C++11
http://dl.dropbox.com/u/13100941...


Based on my experience, GoingNative 2012: All Sessions are now available On-Demand! (Channel 9) and Videos " C++ and Beyond are quite helpful for me to see the best practice of using C++11.

Besides, you can find all proposals at C++0x/C++11 Support in GCC, which would be good references if you are not clear about a specific feature. The best reference would be the ISO standard of C++ (Information technology -- Programming languages -- C++), which will cost 30$ to get it. :( You can find the draft here (JTC1/SC22/WG21 - The C++ Standards Committee), which I have no idea how close it is to the standard version.

Some blogs or rss I am following to get recent news on C++11:

I'll list some books to add to the previous answers.

These books have been updated to include C++11 features, but they attempt to cover the whole C++ language:

* C++ Primer (5th Edition) by Stanley Lippman et al (http://www.amazon.com/Primer-5th...)

* The C++ Standard Library: A Tutorial and Reference (2nd Edition) by Nicolai Josuttis (http://www.amazon.com/Standard-L...)

* Professional C++ by Marc Gregoire (http://www.amazon.com/Profession...)

* C++ Programming Language (4th Edition) by Bjarne Stroustrup will be out next year (http://www.amazon.com/Programmin...)

This book by Anthony Williams focuses only on the concurrency features introduced in C++11:

* C++ Concurrency in Action: Practical Multithreading (http://www.amazon.com/C-Concurre...)

Finally, I've written a book specifically about the C++11 features in Visual Studio 2010:
* C++11 Rocks (http://cpprocks.com)

As far as I know, my book is the only one that focuses specifically on C++11.

I have also compiled a comparison of the C++11 language support in GCC 4.7, Clang 3.1 and VS2012. The feature table is here: http://www.cpprocks.com/a-compar...

As far as library support goes, libc++ should be feature complete, while stdlibc++ still has some features missing, and VS2012's implementation is supposed to be complete with the exception of library features which depend on as yet unimplemented language features (e.g. STL algorithms that take an initializer_list aren't present).
